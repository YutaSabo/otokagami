# Phase 11: モバイル判定体験

## 運用ルール

- ユーザーが「Phase 11 の指示書をもとに作業してください」と指示したら、この指示書を根拠に開発を開始する。
- この指示書に書かれた内容がすべて完了したら、Phase 11 は完了とし、作業を終了する。勝手に Phase 12 へ進まない。
- 次の Phase は、ユーザーが改めて実行を依頼したときに開始する。
- 判定体験は録音/判定①、詳細②、直し方③の3画面に収める。
- 実際のAPIキー、トークン、秘密鍵、service role keyをチャットやコミットに出さない。

## ゴール

デイリー練習の主体験をモバイルアプリで実装する。Phase 11 完了時点では、ホームからデイリー練習を開始し、1問ずつ録音、判定、2段IPA表示、詳細、直し方、再録音、次へを行い、7問完走できる。

## 前提条件

- Phase 7、Phase 8、Phase 10が完了している。
- Next.js APIがローカルで起動できる。
- iPhoneまたはiOS SimulatorでExpo dev clientを起動できる。
- 参照設計書:
  - `docs/specs/02_MVP_SCOPE.md`
  - `docs/specs/03_UX_SPEC.md`
  - `docs/specs/04_CORE_LOGIC.md`
  - `docs/specs/07_API_SPEC.md`
  - `docs/specs/09_PRIVACY_BILLING_SECURITY.md`
  - `docs/specs/10_TEST_PLAN.md`

## スコープ

含む:

- デイリー練習開始。
- `/api/daily-session` 呼び出し。
- 録音/判定①。
- 詳細②。
- 直し方③。
- お手本再生。
- ユーザー録音再生。
- 端末ローカル録音保存。
- 1問内の再録音。
- 7問完走。

含まない:

- 苦手ドリルUI。
- 音素表練習UI。
- 自由入力UI。
- 進捗画面の完成。
- 通知、削除、書き出し。

## 作業タスクリスト

1. デイリー開始フローを実装
   - ホームのスタートボタンから `/api/daily-session` を呼ぶ。
   - 今日未開始なら1問目へ進む。
   - 今日進行中なら次の未完了問題へ進む。
   - 今日完了なら完了状態を表示し、再練習は練習タブに任せる。
   - 8日目以降でPro無効なら課金ウォールを出し、閉じたら開始しない。

2. 録音/判定①を実装
   - 判定前:
     - 戻る。
     - 対象単語または文。
     - 期待IPA。
     - お手本再生。
     - スロー再生。
     - 録音ボタン。
   - 録音中:
     - 録音中状態。
     - 停止ボタン。
     - タブ移動や戻る操作時は録音破棄確認。
   - 判定中:
     - ローディング。
     - 多重送信防止。
     - 通信失敗時の再試行。
   - 判定後:
     - overallスコア。
     - 2段IPA。
     - 緑/黄/赤の音素表示。
     - お手本再生。
     - ユーザー録音再生。
     - もう一度録音。
     - 詳細を見る。
     - 次へ。

3. 録音とストリーミング判定を実装
   - iOSネイティブモジュールでマイク入力を16kHz/16bit/mono PCMへ変換する。
   - バックエンド発行の短期トークンを使い、録音中のPCMをAzure Speech SDKへ直接push streamする。
   - `/api/assess` には音声ではなくAzure結果JSONと性能計測値だけを送信する。
   - `attempt_no` は同一問題内で1から連番にする。
   - ユーザー録音は端末ローカルに保存してよい。
   - サーバーやSupabase Storageにユーザー録音を保存しない。

4. 2段IPA表示を実装
   - 上段は期待IPA。
   - 下段は実測IPA。
   - 音素単位で対応させる。
   - 違う音、または黄/赤の音を目立たせる。
   - 色だけに依存せず、ラベルまたは記号を併用する。

5. 間違いゼロ表示を実装
   - 全音素が緑の場合のみ「素晴らしい！」を表示する。
   - 詳細を強制せず、次へ進みやすくする。
   - 問題の正解判定と間違いゼロ表示を混同しない。

6. 詳細②を実装
   - 対象単語または文。
   - 音素チップ一覧。
   - 各音素のスコア。
   - 期待IPA。
   - 実測IPA。
   - 要修正音。
   - もう少しの音。
   - お手本再生。
   - ユーザー録音再生。
   - 直し方への導線。
   - 文問題ではタップした音素の対象単語にフォーカスする。

7. 直し方③を実装
   - `/api/advice/:advice_id` で助言を取得する。
   - 表示要素:
     - 戻る。
     - 対象音素または混同ペア。
     - 母語: 未指定またはプロフィールの言語タグ。判定スコアには影響しない。
     - コツ。
     - 静的図解アセット。
     - お手本再生。
     - もう一度録音。
     - 助言評価: 役立った/役立たなかった。
   - もう一度録音は同じ問題の録音/判定①へ戻し、同一問題内attemptとして扱う。

8. 助言評価を実装
   - Phase 9で実装済みの `/api/advice-feedback` を呼ぶ。
   - `advice_id` と `attempt_id` を紐付ける。
   - free attempt由来ではない場合、`free_attempt_id` はnull。

9. 次へ/完走を実装
   - ユーザーが次へを押したら次の問題に進む。
   - 7問完了後はホームに戻り、`7 / 7` と完了状態を表示する。
   - 1問内で複数回録音した場合、集計更新はサーバー側best attemptに任せる。

10. E2E確認を作成
   - Azureモックでデイリー完走を確認する。
   - 1問目で録音、判定、詳細、直し方、再録音、次へが動くことを確認する。
   - 全音素80以上モックで「素晴らしい！」が出ることを確認する。

## 動作確認手順

- Supabase local、Python推論サービス、Next.js API、Expo dev clientを起動する。
- 新規ユーザーでホームからスタートする。
- 1問目を録音し、判定後に2段IPAとスコアを確認する。
- 詳細へ進み、赤/黄の音から直し方へ進む。
- 直し方からもう一度録音する。
- 次へ進み、7問完了する。
- ホームで `7 / 7` と完了状態を確認する。
- サーバーやSupabase Storageにユーザー音声が保存されていないことを確認する。
