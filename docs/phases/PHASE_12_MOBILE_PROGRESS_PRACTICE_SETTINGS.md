# Phase 12: モバイル周辺機能

## 運用ルール

- ユーザーが「Phase 12 の指示書をもとに作業してください」と指示したら、この指示書を根拠に開発を開始する。
- この指示書に書かれた内容がすべて完了したら、Phase 12 は完了とし、作業を終了する。勝手に Phase 13 へ進まない。
- 次の Phase は、ユーザーが改めて実行を依頼したときに開始する。
- 自由入力はPro限定で、無料期間中でも利用不可とする。
- 実際のAPIキー、トークン、秘密鍵、service role keyをチャットやコミットに出さない。

## ゴール

モバイルアプリの進捗、練習タブ、自由入力、通知、設定、データ削除、JSON書き出し、購入復元導線を完成させる。Phase 12 完了時点では、ローカル環境でMVP画面一式が受け入れ基準を満たす。

## 前提条件

- Phase 9、Phase 10、Phase 11が完了している。
- Next.js APIがローカルで起動できる。
- 参照設計書:
  - `docs/specs/02_MVP_SCOPE.md`
  - `docs/specs/03_UX_SPEC.md`
  - `docs/specs/04_CORE_LOGIC.md`
  - `docs/specs/07_API_SPEC.md`
  - `docs/specs/09_PRIVACY_BILLING_SECURITY.md`
  - `docs/specs/10_TEST_PLAN.md`

## スコープ

含む:

- 進捗画面完成。
- 練習タブ完成。
- 苦手ドリルUI。
- 音素表練習UI。
- Pro限定自由入力UI。
- 自由入力保存同意モーダル。
- iOSローカル通知。
- 設定画面完成。
- 購入復元導線。
- 学習データ削除。
- JSON書き出し。

含まない:

- 本番環境作成。
- ステージングE2E。
- TestFlight配布。
- Phase 2機能のランキング、週間チャレンジ、成果シェア画像。

## 作業タスクリスト

1. 進捗画面を完成させる
   - `/api/progress` を呼ぶ。
   - 表示要素:
     - 現在ストリーク。
     - 最長ストリーク。
     - カレンダー。
     - 苦手音ヒートマップ。
     - 総合習熟度グラフ。
     - レベル。
     - 称号。
     - 獲得済みバッジの一部。
   - グラフは日/週/月の表示切替を持つ。
   - 指標は総合習熟度。生スコアの日次平均は使わない。
   - `free_attempts` は進捗表示に使わない。

2. 練習タブを完成させる
   - 表示要素:
     - 苦手ドリルを始める。
     - 音素表から選ぶ。
     - 自由入力。
   - 練習開始前に無料期間/Pro状態を確認する。
   - 8日目以降でPro無効なら課金ウォールを表示する。

3. 苦手ドリルUIを実装
   - `/api/practice-session` に `mode = weak_drill` を送る。
   - 返却されたパック問題をPhase 11と同じ判定フローで練習する。
   - `practice_mode = weak_drill` として `/api/assess` に送る。
   - パック問題なのでattempt保存と習熟度更新対象になる。

4. 音素表練習UIを実装
   - `phonemes` のIDリストに基づくグリッドを表示する。
   - 習熟度に応じた色を付けてよい。
   - 音素を選ぶと `/api/practice-session` に `mode = phoneme_select` と `phoneme_id` を送る。
   - 返却された問題をPhase 11と同じ判定フローで練習する。
   - `practice_mode = phoneme_select` として `/api/assess` に送る。

5. 自由入力UIを実装
   - 無料期間中でもProでなければ利用不可。
   - 未Proの場合は課金ウォールを表示する。
   - Pro有効で初回利用時、保存同意モーダルを出す。
   - 同意文では最低限、入力文保存、個人情報を入れないこと、音声非保存、削除対象であることを伝える。
   - 同意後 `/api/free-text-consent` を呼ぶ。
   - 録音して `/api/free-assess` を呼ぶ。
   - 日次ソフトキャップ超過時は翌日案内を表示する。
   - 自由入力結果は進捗やストリークに混ぜない。

6. iOSローカル通知を実装
   - 初回起動直後には通知許可を求めない。
   - 設定で復習リマインダーを有効にするときに許可を求める。
   - 復習リマインダーON/OFFと時刻を保存する。
   - APNsやサーバープッシュは使わない。
   - 通知権限拒否時はリマインダーOFFとして扱う。

7. 設定画面を完成させる
   - プラン状態。
   - Pro登録。
   - 購入復元。
   - 母国語: 日本語。
   - 目標アクセント: US。
   - UK: 準備中または無効表示。
   - 復習リマインダー。
   - リマインダー時刻。
   - 再生速度。
   - 学習データ書き出し。
   - 学習データ削除。
   - プライバシーポリシー。
   - 利用規約。
   - アプリ情報。

8. 購入復元導線を実装
   - RevenueCat SDKの購入復元を設定画面と課金ウォールから呼べるようにする。
   - 復元後、`/api/access-status` またはRevenueCat同期でPro状態を更新する。
   - RevenueCat取得失敗時は購読状態を確認できない旨を表示する。

9. 学習データ削除を実装
   - 削除前に確認モーダルを出す。
   - サーバーデータと端末ローカル録音が削除対象であることを明示する。
   - `/api/delete-learning-data` を呼ぶ。
   - 成功後、端末ローカル録音を削除する。
   - 進捗、ストリーク、レベル、バッジ、称号は初期状態に戻る。
   - 無料期間起点と購読状態は削除しない。

10. JSON書き出しを実装
   - `/api/export` を呼ぶ。
   - 端末の共有/保存UIでJSONを書き出せるようにする。
   - 音声ファイルは含めない。
   - 秘密情報は含めない。

11. 手動確認とE2Eを行う
   - `docs/specs/10_TEST_PLAN.md` のE2E、手動確認、自由入力、データ管理項目をローカルで確認する。

## 動作確認手順

- ローカルAPIとExpo dev clientを起動する。
- 進捗画面でストリーク、ヒートマップ、グラフ、レベル、称号、バッジを確認する。
- 苦手ドリルを開始し、判定フローで練習できることを確認する。
- 音素表から音素を選び、判定フローで練習できることを確認する。
- 無料期間中ユーザーで自由入力が課金ウォールになることを確認する。
- Pro有効ユーザーで自由入力同意、判定、`free_attempts` 保存、進捗非反映を確認する。
- 通知許可拒否時に設定が破綻しないことを確認する。
- データ削除とJSON書き出しを確認する。
