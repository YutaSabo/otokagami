# Pronunciation Mirror 開発仕様書 README

このディレクトリは、Pronunciation Mirror MVP を実装するための開発仕様書一式である。後続の Codex セッションは、この会話を参照せず、このディレクトリ内の仕様書とリポジトリ内ファイルだけを根拠に実行指示書を作成し、実装を進める。

## 仕様の根拠

- 原典: `マスター設計書.md`
- フェーズ2決定事項:
  - iPhone のみを対象にする。
  - Expo + dev client/prebuild 前提で実装する。
  - MVPの判定は国籍・母語を問わず、`en-US`を標準ロケールとする。初期UIと助言は日本語版を先行する。
  - 認証は Supabase anonymous auth、購読管理は RevenueCat、課金は iOS IAP のみ。
  - 7日間はアプリ側管理の無料期間、8日目以降は Pro 登録必須の課金ウォールを表示する。
  - Apple のイントロオファーは使わない。
  - Supabase は Docker 経由でローカル実装する前提で開発環境を構成する。
  - 音声は短期トークンでiPhoneからAzureへ直接ストリーミングし、自社サーバーへ送信・保存しない。
  - 自由入力は Pro 限定で、初回同意後に raw text を保存できる。
  - サーバー専用キーは Expo アプリ側に置かない。
  - 自由入力の同意状態は `profiles.free_text_consent_version` と `profiles.free_text_consented_at` に保存する。
  - 再インストール対策として、端末識別補助IDのハッシュを `installations` に保存し、無料期間の再付与を防ぐ。
  - RevenueCat App User ID は Supabase `auth.users.id` と同じ値に統一する。
  - 苦手ドリルと音素表練習の出題は `POST /api/practice-session` でサーバー側に集約する。
  - コンテンツseedは未レビューを `is_active = false` とし、ユーザーのレビュー通過分のみ `true` にする。
  - Phase指示書はローカル完走を主対象とし、最終Phaseでステージング環境のE2E確認を含める。

## MVPの一文定義

世界の英語学習者が、毎日7個の単語・文を iPhone で録音し、Azure発音評価、IPA、短いローカライズ助言、進捗可視化、苦手ドリルによって「通じない音」を継続的に直すアプリを作る。母語・国籍によるスコア補正は行わない。

## MVPに含めるもの

- デイリー練習: 単語5 + 文2。
- 判定フロー: 録音/判定①、詳細②、直し方③。
- 進捗基礎: ストリーク、ヒートマップ、総合習熟度グラフ。
- 苦手ドリル。
- 音素表から選ぶ練習。
- Pro 限定のテキスト自由入力。
- レベル/バッジ/称号。
- 設定: プラン管理、購入復元、母国語/目標アクセント表示、通知、再生速度、データ削除、JSON書き出し、規約リンク。
- iOS ローカル通知。
- 端末ローカルの録音履歴保存と聞き返し。
- RevenueCat + iOS IAP による Pro 購読。
- Docker 経由のローカル Supabase 開発環境。

## MVPに含めないもの

- Android。
- iPad 専用UI。
- Web お試しページの実装。
- 多言語UI。
- UKアクセントでの判定・出題・TTS。
- AI会話。
- カリキュラム、レッスン動画、解説動画。
- 発音ゲーム。
- 単語帳。
- `en-US`以外のロケール向けprosody判定。
- 自信度表示。
- 効く順ランキング。
- 匿名ランキング。
- 週間チャレンジ。
- 成果シェア画像。
- シャドーイング。
- Stripe 決済。
- 実行時のAI画像生成。

Web お試しページ、Android、多言語UI、Stripe、ランキング、シェア画像は将来拡張または Phase 2 の候補として扱う。シャドーイングは今回の設計書対象から外す。

## 実装時の絶対ルール

- 仕様の根拠は、この仕様書一式と `マスター設計書.md` に限定する。
- 根拠のない仕様を追加しない。
- 未決事項を見つけた場合は `TBD` として明示し、勝手に埋めない。
- サーバー専用キーを Expo アプリ側の環境変数に置かない。
- 実際の API キー、トークン、秘密鍵、service role key をチャットに出力しない。
- 実際の API キー、トークン、秘密鍵、service role key をコミットしない。
- 音声ファイルをサーバーまたは Supabase Storage に保存しない。
- 自由入力の raw text は、初回同意後にのみ保存する。
- 自由入力の結果を `phoneme_state`、ストリーク、復習、ヒートマップへ反映しない。
- Apple のイントロオファーは使わない。
- 8日目以降は自動課金ではなく、Pro 登録必須の課金ウォールを表示する。

## シークレット運用

開発開始時、人間が `.env.example` をコピーして `.env` を作り、必要な値を埋める。Codex は `.env` が埋まっている前提で実装を進める。

Codex は以下を行ってはならない。

- 実際のキー値をユーザーにチャットへ書かせる。
- 実際のキー値を出力する。
- `.env`、`.env.local`、その他秘密情報を含むファイルをコミットする。
- `EXPO_PUBLIC_` 変数に Azure、OpenAI、Supabase service role、RevenueCat secret などのサーバー専用キーを置く。

キーが必要な作業に着手する前に値が未設定であれば、Codex は「`.env` の `VARIABLE_NAME` をローカルで埋めてください」と依頼する。値そのものはチャットに書かせない。

`.env.example` に新しい変数を追加した場合、Codex は新しく人間が埋める必要のある項目名を通知する。

詳細は `08_ARCHITECTURE.md` と `.env.example` を参照する。

## 用語

| 用語 | 意味 |
| --- | --- |
| デイリー練習 | 1日7個の提示問題。単語5、文2。 |
| attempt | 録音1回分の判定ログ。やり直しもすべて保存する。 |
| scored attempt | Azure判定に成功し、スコア保存対象になった attempt。 |
| best attempt | 1問内でターゲット音素平均が最高の attempt。集計更新に使う。 |
| phoneme_state | ユーザーごとの音素習熟度、練習回数、復習日を持つ集計状態。 |
| EWMA | 習熟度の指数移動平均。初回はスコアそのまま、以後 `0.3 * score + 0.7 * old`。 |
| ターゲット音素 | 問題が練習対象としている音素。正解判定と習熟度更新の対象。 |
| 実測音素 | Azure 発音評価 API が返す、ユーザー音声がどの音に聞こえたかの音素。 |
| 混同ペア | 期待音素と実測音素の組み合わせ。例: `r -> l`、`v -> b`。 |
| 直し方ページ | 混同ペアまたは音素に対応する静的な助言・図解ページ。 |
| Pro | iOS IAP の自動更新サブスク。RevenueCatで購読状態を管理する。 |
