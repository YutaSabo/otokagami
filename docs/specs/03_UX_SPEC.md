# UX Spec

## 対象

MVPの対象UIは iPhone アプリのみである。React Native (Expo) で実装する。iPad専用UI、Android、Webは対象外。

## 全体構成

アプリは下タブ4画面と判定フロー3画面で構成する。

下タブ:

- ホーム。
- 進捗。
- 練習。
- 設定。

判定フロー:

- 録音/判定①。
- 詳細②。
- 直し方③。

下タブは原則常時表示する。録音中、課金ウォール、同意モーダル、データ削除確認など、操作を中断すべき場面ではモーダルを使ってよい。

## 共通表示ルール

### 色

音素スコアの色は全画面共通とする。

| 色 | 条件 | 意味 |
| --- | --- | --- |
| 緑 | 80以上 | 通じる範囲。 |
| 黄 | 60以上80未満 | 惜しい。 |
| 赤 | 60未満 | 要改善。 |

しきい値は画面ごとに変えない。

### 2段IPA

判定後は、期待IPAと実測IPAを2段で表示する。

- 上段: 期待IPA。
- 下段: 実測IPA。
- 音素単位で対応させる。
- 違う音、またはスコアが黄/赤の音を目立たせる。

Azureの返却が単語単位や音素単位でずれる場合は、API層で正規化し、UIは正規化済みの `phoneme_results` を描画する。

### テキスト量

- 録音/判定①では説明を増やさない。
- 詳細情報は詳細②に逃がす。
- 直し方の本文は1〜2行を基本とし、最大でも3行まで。
- 自信度、効く順ランキング、長い解説は出さない。

### 背景

- 録音/判定①はダーク基調を既定にする。
- 詳細②と直し方③は読みやすい白基調を既定にする。
- 実装時の色・余白・タイポグラフィは調整可能だが、情報量と遷移構造は変更しない。

## 初回起動

### 初期化

初回起動時に行うこと:

1. Supabase anonymous auth で匿名ユーザーを作成する。
2. サーバー側にユーザープロファイルを作成する。
3. `native_language = ja` を保存する。
4. `target_accent = US` を保存する。
5. `free_trial_started_at` をサーバー時刻で保存する。
6. Keychain等に再インストール耐性のある端末識別補助IDを保存する。

初回起動でメール登録やAppleログインを要求しない。

### オンボーディング

MVPでは最小限でよい。

- アプリの目的を1〜2画面で示す。
- 7日間は無料で使えることを示す。
- 自由入力はPro限定であることを示す。
- 8日目以降はPro登録が必要であることを示す。
- 通知許可は、初回起動直後ではなく、復習リマインダーを有効にするときに求める。

## 課金ウォール

### 表示条件

次の条件をすべて満たす場合、練習開始前に課金ウォールを表示する。

- `free_trial_started_at` から8日目以降である。
- RevenueCat上でPro entitlementが有効ではない。
- ユーザーがデイリー練習、苦手ドリル、音素表練習、自由入力のいずれかを開始しようとしている。

### 表示内容

- 月額580円。
- 年額4,980円。
- Pro登録ボタン。
- 購入復元ボタン。
- 利用規約とプライバシーポリシーへのリンク。

Appleイントロオファーは使わない。表示上も「Apple側の無料トライアル」は訴求しない。

### 閉じた場合

Pro未登録のまま課金ウォールを閉じた場合、練習は開始しない。ホームまたは設定に戻す。

## ホーム

### 目的

今日の練習の入口に徹する。

### 表示要素

- アプリ名。
- 現在のストリーク。
- 今日の進捗 `n / 7`。
- 大きなスタートボタン。
- 7個完了後の完了状態。

### 表示しないもの

- 今日の7個の一覧。
- 長い説明。
- 詳細な苦手音一覧。
- 自由入力欄。

### 状態

| 状態 | 表示/動作 |
| --- | --- |
| 今日未開始 | `0 / 7` とスタートボタンを表示。 |
| 今日進行中 | 現在の完了数を表示。スタートで次の未完了問題へ進む。 |
| 今日完了 | 完了表示、ストリーク、進捗への導線を表示。再練習は練習タブから行う。 |
| Pro期限切れ | 課金ウォール導線を表示。練習開始時は課金ウォール。 |
| オフライン | 練習開始不可。既存履歴閲覧への導線を表示。 |

## 録音/判定①

### 目的

1問を提示し、録音、判定、聞き比べ、詳細導線を提供する。

### 判定前の表示

- 戻る。
- 対象の単語または文。
- 期待IPA。
- お手本再生。
- スロー再生。
- 録音ボタン。

判定前にスコアを出さない。

### 録音中の表示

- 録音中状態。
- 停止ボタン。
- 入力レベルの簡易表示は任意。
- タブ移動や戻る操作時は録音破棄の確認を出す。

### 判定中の表示

- 判定中ローディング。
- 多重送信を防ぐ。
- 通信失敗時は再試行を提示する。

### 判定後の表示

- overallスコア。
- 2段IPA。
- 緑/黄/赤の音素表示。
- お手本再生。
- ユーザー録音再生。
- もう一度録音。
- 詳細を見る。
- 次へ。

### 次へ

ユーザーが満足した場合、次の問題へ進む。1問内で複数回録音した場合、集計更新にはbest attemptを使う。

### 間違いゼロ

全音素が緑の場合は、詳細を強制せず「素晴らしい！」を表示し、次へ進みやすくする。

## 詳細②

### 目的

単語または文内の音素ごとの出来を確認し、直すべき音へ誘導する。

### 表示要素

- 戻る。
- 対象の単語または文。
- 音素チップ一覧。
- 各音素のスコア。
- 期待IPA。
- 実測IPA。
- 要修正音。
- もう少しの音。
- お手本再生。
- ユーザー録音再生。
- 直し方への導線。

### 文の扱い

文問題で音素をタップした場合、対象単語と該当音素にフォーカスする。文全体を詳細化しすぎない。

### 直し方への遷移

赤または黄の音素をタップすると、該当する混同ペアの直し方③へ進む。混同ペアページがない場合は汎用ページへ進む。

## 直し方③

### 目的

間違った音だけに対して、日本語の短い助言と静的図解で再録音につなげる。

### 表示要素

- 戻る。
- 対象音素または混同ペア。
- 母国語: 日本語。
- コツ。
- 口・舌の静的図または短いアニメーション風アセット。
- お手本再生。
- もう一度録音。
- 助言評価: 役立った/役立たなかった。

### 助言評価

助言評価は、表示した `advice_id` と紐付けて保存する。これは助言の効果検証に使う。

### 再録音

もう一度録音する場合、同じ問題の録音/判定①へ戻る。同一問題内のattemptとして扱う。

## 進捗

### 目的

追加コストなしで、継続価値を表示する。

### 表示要素

- 現在ストリーク。
- 最長ストリーク。
- カレンダー。
- 苦手音ヒートマップ。
- 総合習熟度グラフ。
- レベル。
- 称号。
- 獲得済みバッジの一部。

### 集計元

進捗は `attempts`、`phoneme_state`、`phoneme_snapshots`、`user_badges` から作る。

`free_attempts` は使わない。

### グラフ

- 日/週/月の表示切替を持つ。
- 指標は総合習熟度。
- 総合習熟度は全音素EWMAの平均。
- 生スコアの日次平均は使わない。

## 練習

### 目的

デイリー以外の練習入口を置く。

### 表示要素

- 苦手ドリルを始める。
- 音素表から選ぶ。
- 自由入力。

### 苦手ドリル

`phoneme_state` のEWMAが低い音素を優先して出題する。デイリーと同じ判定フローを使う。

### 音素表

音素IDリストに基づくグリッドを表示する。各音素には、習熟度に応じた色を付けてよい。

### 自由入力

- Pro限定。
- トライアル中も利用不可。
- 未Proの場合は課金ウォールを表示する。
- 初回利用時、入力文保存に関する同意を求める。
- 1日20回程度のソフトキャップを超えた場合、翌日案内を表示する。

## 設定

### 表示要素

- プラン状態。
- Pro登録。
- 購入復元。
- 母国語: 日本語。
- 目標アクセント: US。
- UK: 準備中または無効表示。
- 復習リマインダー。
- リマインダー時刻。
- 再生速度。
- 学習データ書き出し。
- 学習データ削除。
- プライバシーポリシー。
- 利用規約。
- アプリ情報。

### データ削除

削除前に確認モーダルを出す。削除対象にサーバーデータと端末ローカル録音が含まれることを明示する。

### JSON書き出し

書き出しは簡易版でよい。対象には練習ログ、音素状態、バッジ、設定、自由入力ログを含める。ただし音声ファイルは含めない。

## オフライン時UX

MVPはオンライン必須である。

| 操作 | オフライン時の動作 |
| --- | --- |
| デイリー練習開始 | 不可。オンラインが必要と表示。 |
| 録音/判定 | 不可。Azure判定が必要と表示。 |
| 自由入力 | 不可。 |
| 履歴閲覧 | 可能。端末ローカルの録音と保存済み結果を表示。 |
| 設定閲覧 | 可能。ただし購読状態更新や購入復元は不可。 |

## エラー時UX

| エラー | 表示/動作 |
| --- | --- |
| Azure判定失敗 | 再試行ボタンを表示。練習回数には入れない。 |
| TTS生成失敗 | お手本生成に失敗したことを表示し、再試行。 |
| OpenAI助言失敗 | テンプレまたは汎用助言にフォールバック。 |
| RevenueCat取得失敗 | 購読状態を確認できないと表示。練習開始時は安全側で課金状態確認を再試行。 |
| Supabase保存失敗 | ローカルに一時状態を保持し、再試行を促す。集計更新済みと見なさない。 |
| 通知権限拒否 | リマインダーOFFとして扱う。設定から再案内。 |

## アクセシビリティ

- 録音、再生、次へ、戻る、購入、削除など主要操作は十分なタップ領域を持つ。
- 色だけに依存せず、緑/黄/赤に対応するラベルまたは記号を併用する。
- 音素記号は読みやすいサイズにする。
- 動的文字サイズへの対応を考慮する。

## フェーズ4セルフレビュー

- マスター設計書・既作成ドキュメントとの矛盾: 3画面フロー、下タブ4画面、7日後課金ウォール、自由入力Pro限定、シャドーイング除外を反映済み。
- Codexが実装に着手できる具体性: 各画面の表示要素、状態、遷移、エラー時挙動を明記済み。
- 用語・命名の一貫性: `daily_session`、`phoneme_results`、`attempts`、`free_attempts`、`advice_id` を後続仕様と接続する形で使用。
