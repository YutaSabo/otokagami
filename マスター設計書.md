# Pronunciation Mirror MVP設計

利用者: me
種別: 設計

<aside>
🪞

**Pronunciation Mirror（発音ミラー）** — 毎日7個の英単語・英文を話すだけ。基準発音との近さを**音素ごとに色とスコアで**見せ、改善対象だけを短い助言と図解で示す。
**世界の英語学習者向け × 音素単位の発音ミラー × 超シンプルな習慣化アプリ。**

</aside>

## 🧭 概要

**英語学習者の伝わりにくい音を、毎日7個の短い練習で直す鏡アプリ。**

- **何を**：提示された英単語・英文を録音 → ネイティブに聞こえた音を音素ごとに色で表示 → 外した音だけ日本語で直し方を提示。
- **誰に**：国籍・母語を問わない英語学習者。初期標準ロケールは`en-US`とし、ロケール別capabilitiesで将来拡張する。
- **どう違うか**：会話・カリキュラム・動画を足さず、毎日7個の短い練習と低価格サブスクで"続く発音改善"に一点突破する。

## 🎯 ターゲット / 解決する課題

- **ターゲット**：世界の英語学習者。母語別・国籍別のスコア補正は行わない。
- **解決する課題**：L/R、TH、V/B、母音長、語末子音、子音連結などで、本人の意図と違う音・違う単語として聞こえてしまう問題。
- **狙う成果**：「ネイティブそっくり」ではなく、相手に誤解されず**通じる音**を増やす。

ユーザーの悩みは「英語っぽく聞こえない」よりも、「自分の発音が相手に何の音として聞こえているか分からない」ことにある。単語全体の点数だけでは直す場所が分からないため、Pronunciation Mirror は音素ごとに見せ、外した音だけを短く直す。

## 🏆 ポジショニング / 差別化（勝ち筋）

- **立ち位置**：「狭く・深く・習慣化・低価格」を束ねた、世界の英語学習者向け発音矯正アプリ。初期UIと助言コンテンツは日本語版を先行する。
- **勝ち筋**：
    - 毎日7個だけの提示問題で、入力の手間を減らして続けやすくする。
    - 音素単位で「期待IPA」と「実測IPA」を並べ、違う音だけ色で見せる。
    - 日本語話者が外しやすい音に特化した、短い日本語助言と図解を出す。
    - ストリーク、復習、ヒートマップ、推移グラフ、レベル/バッジ/称号で、単発判定ではなく継続改善にする。
    - 判定エンジンを自作せず Azure 発音評価 API を使い、プロダクトの差別化を「効く直し方」に集中させる。
- **本当の堀**：判定そのものではなく、再録音スコア改善とユーザー評価で検証された、日本語話者向けの"効く直し方"の蓄積。

## ✨ 提供価値（コア体験）

- メイン体験は **毎日7個のデイリー練習（単語5＋文2）**。ホームからスタートし、1個ずつ話すだけで進む。
- 各問に🔊お手本音声、録音、Azureによる即時判定、お手本 vs 自分の聞き比べを用意する。
- 判定後は、期待IPAと実測IPAを2段で表示し、音素ごとに緑・黄・赤で「どの音がどう聞こえたか」を示す。
- 外した音は、1〜2行の日本語助言と、事前生成または手作りの静的な口・舌の図解で直す。
- 全音素が緑なら詳細説明を出さず、「素晴らしい！」で次へ進みやすくする。
- 練習結果は習熟度・復習・進捗表示に変換し、毎日続ける理由を作る。

## 🧩 機能一覧

### MVP

| 機能 | 何ができるか | 狙い |
| --- | --- | --- |
| デイリー練習 | 毎日、単語5個＋文2個を1問ずつ出題する。ホームには7個の一覧を出さない。 | 迷わず始め、7個で「今日は終わり」と思える習慣を作る。 |
| 自動出題 | 苦手3、新規2、復習2の固定配分で今日の7個を作る。当日初回生成時に固定し、同じ日は再生成しない。 | 苦手克服、新規開拓、忘却防止のバランスを取る。 |
| 録音と判定 | バックエンドがAzure短期トークンを発行し、iPhoneが録音中のPCMをAzureへ直接ストリーミングして音素単位で判定する。 | 音声ファイル中継と変換待ちを排除し、Subscription Keyを端末へ露出しない。 |
| 2段IPA表示 | 上段に期待IPA、下段に実測IPAを表示し、違う音や基準未満の音を色で示す。 | 「何が違ったか」を単語全体ではなく音単位で理解させる。 |
| 色分け | 緑=80以上、黄=60以上80未満、赤=60未満を全画面共通で使う。 | 画面ごとに解釈が変わらないシンプルなフィードバックにする。 |
| 詳細画面 | 音素ごとの一致度、期待IPA、実測IPA、要修正音、聞き比べを表示する。 | メイン画面を重くせず、必要な時だけ原因を見られるようにする。 |
| 直し方ページ | 混同ペアまたは音素に対応する短い日本語助言と静的図解を出す。 | 外した音だけ、すぐ再録音できる粒度で直す。 |
| お手本音声 | Piperでサーバー側生成し、キャッシュする。標準/スローを使う。 | お手本を事前同梱せず、パック差し替えや自由入力にも対応する。 |
| 聞き比べ | お手本音声と端末ローカルのユーザー録音を並べて再生する。 | 自分の音とコーチ音声の差を耳で確認する。 |
| 進捗 | ストリーク、カレンダー、苦手音ヒートマップ、総合習熟度グラフを表示する。 | 判定結果を継続価値に変える。 |
| レベル/バッジ/称号 | 累計完了数、ストリーク、特定音素の改善などで表示を更新する。 | 追加コストなしで達成感を作る。 |
| 苦手ドリル | 習熟度が低い音素を優先して連続出題する。 | 進捗画面で見えた苦手をすぐ練習できるようにする。 |
| 音素表から選ぶ練習 | 音素グリッドから練習したい音を選び、該当パック問題を出す。 | ユーザーが自分で課題音を選べる入口を作る。 |
| Pro限定の自由入力 | Pro登録後だけ、任意の英文を入力して判定できる。初回に保存同意を取る。 | 実際に言いたい文を試せる価値を作りつつ、集計の品質とコストを守る。 |
| 復習リマインダー | 次回復習日とユーザー設定時刻をもとに、iOSローカル通知を出す。 | サーバープッシュなしで復習習慣を促す。 |
| 録音履歴 | ユーザー録音は端末ローカルに保存し、聞き返せる。サーバーには保存しない。 | プライバシーを守りながら聞き比べ価値を出す。 |
| データ管理 | 学習データ削除とJSON書き出しを設定から実行できる。 | ユーザーが自分のデータを管理できるようにする。 |
| 課金 | 7日間のアプリ側無料期間後、Pro未登録なら練習開始前に課金ウォールを表示する。 | 無料原価を限定し、サブスク継続で事業化する。 |

## 📱 画面構成 / UX

アプリ全体は **下タブ4画面（🏠ホーム・📊進捗・🎯練習・⚙️設定）＋ 判定フロー3画面（録音/判定①→詳細②→直し方③）**。判定体験のコアは①→②→③で、下タブは原則常時表示する。

録音中、課金ウォール、自由入力同意、データ削除確認など、操作を中断すべき場面ではモーダルを使う。

### 初回起動 / オンボーディング

- メール登録やAppleログインを要求せず、Supabase anonymous authで開始する。
- 初回起動時に、母語未指定（`und`）・USアクセント・無料期間開始日時・タイムゾーンを保存する。母語や国籍でスコアを補正しない。
- 7日間は自由入力を除くMVP機能を無料で使えること、8日目以降はPro登録が必要なことを短く示す。
- 通知許可は初回起動直後ではなく、復習リマインダーを有効にするときに求める。
- 再インストールで無料期間が何度も付与されないよう、端末側の補助IDをハッシュ化してサーバー側で扱う。

### アプリ全体の構成（下タブ）

| タブ | 役割 |
| --- | --- |
| 🏠 ホーム | 今日の練習の入口。ストリーク＋当日の進捗(n/7)＋大きな「スタート」だけ。7個の一覧は出さず、スタート→1個ずつ出題。 |
| 📊 進捗 | ストリークカレンダー・苦手音ヒートマップ・総合習熟度グラフ（日/週/月）・レベル・称号・バッジ。 |
| 🎯 練習 | 苦手ドリル、音素表から選ぶ練習、Pro限定のテキスト自由入力。 |
| ⚙️ 設定 | プラン管理、購入復元、母国語/目標アクセント表示、通知、再生速度、データ削除、JSON書き出し、規約。 |

### 判定フロー（コア3画面）

判定体験は **「3画面」だけ**（①録音/判定 → ②詳細 → ③直し方）。

| 画面 | 背景 | 役割・主な要素 |
| --- | --- | --- |
| ① 録音/判定 | ダーク基調 | 今日の7個から1問を表示。お手本、スロー再生、録音、判定、スコア、2段IPA、聞き比べ、詳細導線。 |
| ② 詳細 | 白基調 | 音素ごとの一致度、期待IPA、実測IPA、要修正音、聞き比べ、直し方への導線。 |
| ③ 直し方 | 白基調 | 混同ペアまたは音素に対応する短い助言、2音の比較、静的図解、お手本、再録音、助言評価。 |

**表示ルール**

- 情報を足したくなったら画面②③へ逃がし、録音/判定①は重くしない。
- 色の意味は3段（緑＝通じる範囲／黄＝惜しい／赤＝要改善）。しきい値は全画面共通。
- 問題の正解はターゲット音素すべてが80以上。間違いゼロ表示は全音素が80以上。
- 間違いゼロなら詳細を強制せず、「素晴らしい！」で次へ進みやすくする。
- 解説は1音につき1〜2行を基本とし、最大でも3行まで。
- 自信度表示・効く順ランキングは入れない。
- 色だけに依存せず、ラベルや記号も併用する。

（各画面の詳細ワイヤーフレームは下記トグル。）

    ```
    ┌──────────────────────────┐
    │  発音ミラー        🔥 7日連続  │
    │                            │
    │        今日の練習           │
    │   ███████░░░░░░  3 / 7    │
    │                            │
    │     ╭──────────────╮       │
    │     │   ▶ スタート    │       │
    │     ╰──────────────╯       │
    │                            │
    │   タップで1個ずつ出題されます  │
    │                            │
    ├──────────────────────────┤
    │ 🏠ホーム  📊進捗  🎯練習  ⚙️設定 │
    └──────────────────────────┘
    ```
    
    - 情報を最小化：ストリーク＋当日進捗(n/7)＋大きな「スタート」。
    - **7個の一覧は出さない。**

    - 判定後はスコア、2段IPA、お手本↔自分の聞き比べ、詳細導線、もう一度録音、次への導線を出す。
    - Azure判定失敗時は再試行を提示し、練習回数・ストリーク・習熟度には入れない。

## ⚙️ コアロジック / アルゴリズム

### 出題（今日の7個 = 単語5＋文2）

- **素材**：全音素40前後（MVP定義では子音24、単母音12、二重母音5）と、`str`、`spl`、`kt` などの子音連結グループを扱う。
- **パック**：各音素に単語パック・文パックを用意し、ターゲット音素、日本語話者難易度、USアクセント、期待IPAを持たせる。
- **レビュー**：AIで事前生成してよいが、未レビューのパックや直し方ページは本番出題対象にしない。レビュー済みだけを有効化する。
- **当日固定**：今日の7個は当日初回生成時に固定し、同じローカル日付では再生成しない。

**配分（3枠固定）**

| 枠 | 中身 | 個数 | 選び方 |
| --- | --- | ---: | --- |
| 🔴 苦手 | 単語2＋文1 | 3 | 習熟度が低い音素を優先。同点は日本語話者難易度が高い音を優先。 |
| 🟡 新規 | 単語2 | 2 | 練習回数0の音素を優先。同点は日本語話者難易度が高い音を優先。 |
| 🟢 復習 | 単語1＋文1 | 2 | 次回復習日が来た音素を優先。 |

**フォールバック**

- 新規が尽きたら復習に回す。
- 復習候補が足りなければ苦手候補で埋める。
- 全体が安定している場合は、苦手枠も復習中心のメンテナンスにする。
- パック問題が不足する場合は、未出題を優先したうえで、最後は重複を許可する。

### 判定

- 判定は **Azure 発音評価 API** を使う。
- 音声は短期トークンでiPhoneからAzureへ直接ストリーミングし、自社サーバー、DB、Supabase Storageには送信・保存しない。
- 保存するのは、音声本体を除く判定レスポンス、音素ごとの期待音・実測音・スコア、正解判定、集計に必要な情報。
- `en-US`ではprosody、miscue、IPA、最大5件の候補音素、音節、Offset/Durationを取得し、取得不能なロケールではcapabilitiesとnullで明示する。
- Azure成功時だけ練習ログとして保存する。通信失敗、Azure失敗、保存失敗は練習回数・ストリーク・習熟度に入れない。

### 正解判定と表示

| 判定 | 条件 | 使い道 |
| --- | --- | --- |
| 問題の正解 | ターゲット音素すべてが80以上 | 復習間隔や習熟度更新に使う。 |
| 間違いゼロ | 全音素が80以上 | 「素晴らしい！」表示に使う。 |
| 緑 | スコア80以上 | 通じる範囲。 |
| 黄 | 60以上80未満 | 惜しい。 |
| 赤 | 60未満 | 要改善。 |

文問題でも、正解判定はターゲット音素を見る。非ターゲット音素の低スコアだけで、その問題を不正解扱いしない。

### 習熟度と復習

- 録音1回ごとの判定結果は生ログとして残す。
- 同じ問題で複数回録音した場合、集計に使うのはターゲット音素平均が最も高い **best attempt** だけ。
- 音素ごとの習熟度は、直近スコアをそのまま平均するのではなく、EWMA（指数移動平均）で更新する。これにより、短期のブレを抑えながら改善傾向を見られる。
- 習熟度、練習回数、最終練習日、次回復習日は音素ごとに持つ。
- 復習間隔は **1→3→7→14日** の4段階。正解なら次の段階へ進み、不正解なら翌日復習に戻す。
- 子音連結は出題タグとして扱い、習熟度の基本更新は音素単位で行う。

### 集計・可視化ロジック

- 推移グラフの指標は **総合習熟度**。評価済み音素のEWMA平均を使い、未評価音素を0点扱いしない。
- 日/週/月表示では、期間内スナップショットまたは期末値を使う。MVPでは期末値を優先する。
- ストリークは、ローカル日付で判定成功済みのパック練習が1件以上ある日を練習日として数える。
- ヒートマップは音素ごとの習熟度で色付けする。
- レベルは、集計対象になった問題完了数で決める。自由入力や録音回数そのものではなく、best attemptで完了した問題数を見る。
- バッジは、初完走、3日/7日/14日連続、初パーフェクト、TH/RL/VB突破、30問/100問達成など、条件が明確で追加コストのないものをMVPに含める。
- 称号は1つだけ表示し、ストリークや特定音素の状態から優先順位で決める。

### 自由入力の扱い

<aside>
🚫

**自由入力は保存するが、集計には入れない。**  
自由入力の結果は、習熟度・復習・ヒートマップ・ストリーク・レベル・バッジ・称号に一切反映しない。

</aside>

- 自由入力はPro限定。7日無料期間中でも利用不可。
- 初回利用時に、入力文を保存することへの同意を取る。
- 保存するのは、入力文、正規化文、IPA変換結果、OOV語、変換確信度、判定結果、PII疑いフラグ、同意バージョンなど。
- ユーザー音声ファイルは自由入力でもサーバー保存しない。
- Proでも1日20回程度のソフトキャップを設ける。
- 自由入力は、将来の需要マイニング、辞書ギャップ改善、教材パック改善の材料として貯める。ただし自動でデイリーパック化しない。

## 🗄 データ方針

この文書では、DBのテーブル定義ではなく、**何を記録し、何に使うか**を方針レベルで定義する。

**3原則**

1. 録音1回ごとの判定結果は生ログとして残す。ただし、習熟度や復習に使うのは各問題のbest attemptだけ。
2. 集計は音素単位で行う。総合習熟度は音素ごとの習熟度から算出する。
3. 自由入力は専用ログとして保存し、デイリー練習の集計とは完全に分ける。

**保存する主な情報**

| 分類 | 保存するもの | 使い道 |
| --- | --- | --- |
| ユーザー基本情報 | 匿名ユーザーID、任意の母語コード、USアクセント、無料期間開始日時、タイムゾーン、通知設定、再生速度 | 初期化、アクセス制御、設定表示。母語は判定スコアに使わない。 |
| 端末識別補助 | 再インストール耐性のある補助IDのハッシュ | 無料期間の再付与防止 |
| パック練習ログ | 練習日時、問題、attempt番号、音素ごとの期待音/実測音/スコア、overall、target平均、Azureレスポンス（音声本体除く） | 履歴、詳細表示、集計更新 |
| 音素状態 | 音素ごとの習熟度、練習回数、最終練習日、次回復習日、復習段階 | 出題、復習、ヒートマップ |
| 日次スナップショット | 音素ごとの日次習熟度 | 推移グラフ、週/月表示 |
| バッジ/称号用データ | 達成済みバッジ、累計完了数、ストリーク | 達成感、継続理由 |
| ブックマーク | 保存した音素、問題、自由文 | 見返し、再練習 |
| 自由入力ログ | 入力文、正規化文、IPA変換、OOV、判定結果、同意情報、PII疑い | 将来の教材改善、辞書改善 |
| 助言評価 | 表示した助言と👍/👎 | 直し方ページの効果検証 |
| お手本音声キャッシュ | Piperで生成したお手本音声 | TTSコスト削減 |
| エラーログ | 判定、TTS、課金、保存などのエラー概要 | 品質改善、障害調査 |

**保存しないもの**

- ユーザー音声ファイルのサーバーコピー
- Supabase Storage上のユーザー音声
- Azureへ送信した一時音声の永続コピー
- APIキー、秘密トークン、Authorizationヘッダー
- クライアント側に置くべきでないサーバー専用キー

ユーザー音声は、短期トークンを使ってiPhoneからAzureへ直接ストリーミングする。自社バックエンドは音声を受信しない。端末ローカルには聞き返し用WAVを保存してよく、学習データ削除時には削除対象にする。
